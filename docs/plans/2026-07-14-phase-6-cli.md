# Phase 6 — the `puddle` CLI: serving switch, SSH bootstrap, tunnel, distribution

## Context

Phase 6 is what makes puddle usable by other people: today the daemon serves the embedded UI itself on 7433 and the `packages/cli` package is a 3-line stub. This phase delivers `puddle start` / `puddle connect user@host` — the CLI serves the web UI at a stable local origin (`localhost:7433`), reverse-proxies `/api` + `/ws` + `/proxy` to the daemon (directly locally, through an SSH tunnel remotely), bootstraps/updates the daemon on any host from self-contained tarballs, and runs the protocol handshake (SPEC §6, §10, §14 Phase 6). The daemon becomes a headless API server on `127.0.0.1:7434`. Builds must be as small and portable as possible; the CLI is a thin bin over `src/lib/` so a future Electron shell can reuse everything.

**Locked decisions** (user-confirmed 2026-07-14):

- Full release pipeline ships: `build-tarball.mjs` + `install.sh` + `.github/workflows/release.yml` (4-platform matrix, GitHub Release + SHA256SUMS). npm publish job included but gated on an `NPM_TOKEN` secret.
- npm package name **`@puddle-code/cli`** (the user registered the `puddle-code` org; unscoped `puddle` is taken by an unrelated 2014 package). The bin command stays `puddle`. The daemon is deliberately NOT published to npm — tarballs are its only distribution, preserving the no-host-Node guarantee.
- **No protocol bump** — the serving switch touches no wire shape (port default is deployment config; `?host=`/`#token=` are browser conventions; Host/Origin guard already ignores ports). `PROTOCOL_VERSION` stays `{major: 5, minor: 1}`.
- Runtime deps of the CLI: **`ws` only** (bundled at publish); hand-rolled arg parsing, node:http proxy, browser-open. No commander/open/http-proxy.
- Linux floor: glibc (Ubuntu 22.04+ / Debian 12+ / RHEL 9+); Alpine/musl explicitly unsupported with a clear `install.sh` refusal. Windows client: supported but degraded (no ControlMaster — see B4).

**Verified facts (do not re-derive):**

- node-pty 1.1.0 has **no Linux prebuilds** (node-gyp builds `build/Release/pty.node`); darwin prebuilds live in `prebuilds/<os>-<arch>/`. Loader probes `../build/Release`, `../build/Debug`, `../prebuilds/<platform>-<arch>` relative to `lib/` — package shape must be preserved, cannot be bundled.
- better-sqlite3 12.11.1 loads via `require('bindings')('better_sqlite3.node')` → `build/Release/`; runtime deps `bindings` + `file-uri-to-path` only.
- `ws` does guarded `require('bufferutil')`/`require('utf-8-validate')` — mark external in any ESM bundle and add a `createRequire` banner.
- `GET /api/version` is behind `bearerAuth` (`packages/daemon/src/http/app.ts:77-78`) — the CLI must read `~/.puddle/token` **before** the handshake.
- `loadConfig` writes defaults back to `config.json`, so every existing install's file literally says `"port": 7433` — migration needed, not just a default change.
- A pre-Phase-6 `puddled --version` **starts a daemon** (no flag handler) — never probe by execution without the `readlink ~/.puddle/bin/current` check first, and always with a timeout.
- npm names: `puddle` taken, `puddle-cli` and `puddled` free.

## Workstream A — daemon-side serving switch

1. **Port default 7433 → 7434**: `packages/shared/src/api/config.ts` (`port` default only; no new schema field, so `/api/config` stays wire-identical).
2. **Config migration** in `packages/daemon/src/config.ts`: on read of raw JSON before zod parse — if `raw.configVersion === undefined && raw.port === 7433`, delete `raw.port` (falls to new default). `loadConfig`/`saveConfig` persist `configVersion: 2` **in the file only** (zod strips it), so the migration never re-fires and a deliberate post-migration 7433 is respected.
3. **`--version` flag** in `packages/daemon/src/index.ts`: print `puddled <appVersion> (protocol <major>.<minor>)` and exit 0 **before any startDaemon work**. Version comes from `__PUDDLED_VERSION__` esbuild define with a `readFileSync('../package.json')` fallback (ambient `declare const`) so dev `tsc` builds keep working.
4. **Drop embedded assets**: remove `assetsDir` from `packages/daemon/src/index.ts`, `daemon.ts` (`DaemonOptions`), `http/app.ts` (catch-all at line 111); **delete** `packages/daemon/src/http/static.ts` and `packages/daemon/scripts/copy-web-assets.mjs`; daemon build becomes `tsc -b`. Unmatched routes now 404 (correct for a headless API). Update daemon tests that pass `assetsDir`.
5. **Vite dev proxy retarget**: `packages/web/vite.config.ts` `/api` + `/ws` targets 7433 → 7434. `pnpm dev` stays daemon+vite; the CLI is not in the dev loop.

## Workstream B — the CLI (`packages/cli`)

Thin bin over lib. **No `process.exit`/`console.*`/TTY access inside `lib/`** — injectable `Logger`, injectable streams, `CliError {code, message, hint?}`; only `src/index.ts` + `src/cli/` touch the process. This is the Electron seam. Layout:

```
src/index.ts            # shebang, dispatch, signal handling, exit codes
src/cli/{args,run,output}.ts        # argv → Command union (pure); dispatcher owns Ctrl-C; Logger impl
src/lib/types.ts        # Logger, CliEvent, HostTarget, CliError
src/lib/paths.ts        # client-side ~/.puddle (PUDDLE_HOME ?? ~/.puddle)
src/lib/version.ts      # CLI version + pinned daemon version + repo slug (esbuild defines w/ fallbacks)
src/lib/transport/{transport,local,ssh}.ts   # exec/readFile/platform interface; local fs+child_process; ssh master
src/lib/tunnel.ts       # ssh -N -L child, health, reconnect backoff 500ms→10s, close()
src/lib/bootstrap.ts    # installedVersion (readlink current), install/upgrade via embedded install.sh over Transport
src/lib/daemon-client.ts# token+port discovery, GET /api/version /api/sessions /api/host
src/lib/handshake.ts    # decideHandshake() pure + runHandshake() incl. auto-upgrade
src/lib/serve/{ui-server,static,proxy,guard}.ts   # node:http UI server (see B2)
src/lib/start.ts, connect.ts   # → RunningCockpit {origin, browserUrl, daemon, onEvent, stop}
src/lib/attach.ts, ws-client.ts, status.ts, logs.ts, upgrade.ts, browser.ts, net.ts
```

**B1 — commands**: `start [--port] [--no-browser] [--tarball]`, `connect user@host [--port] [--remote-port] [--no-upgrade] [--no-browser] [--tarball]`, `status [host]`, `attach [host] <session> [--term]`, `logs [host] [session] [--term] [-f]`, `upgrade [host]`. Two positionals → first is host; one → local. No `@`-sniffing.

**B2 — serve + proxy** (`lib/serve/`): one `node:http` server on `127.0.0.1`, default 7433, auto-pick next free port on `EADDRINUSE` (explicit `--port` busy = hard error). Routing: `/api`, `/ws`, `/proxy/*` → proxy to a mutable `{host, port}` target ref (daemon port locally, tunnel port remotely; reconnects can repoint); everything else → static. Guards: copy `isLocalHostHeader`/`isLocalOrigin` predicates from `packages/daemon/src/security/middleware.ts:18-34` verbatim. Static: port of the deleted `packages/daemon/src/http/static.ts` to a plain `(req,res)` handler (same MIME table, path confinement, SPA fallback), root `dist/public`. HTTP proxy: verbatim byte-for-byte path/query, headers minus hop-by-hop (reuse the daemon's `HOP_BY_HOP` set from `proxy/http.ts`) but **`Authorization`/`Cookie` pass through** (unlike tier-2 — the daemon needs them), streamed both ways, connect failure → 502 in the daemon's error envelope (`daemon_unreachable`). WS upgrade: single `server.on('upgrade')` splice per `packages/daemon/src/proxy/upgrade.ts` (guard → `http.request` with handshake headers verbatim → `socket.unshift(head)` → pipe both ways); copy `ProxySocketTracker` (`proxy/sockets.ts`) for clean `stop()`. Covers `/ws` and `/proxy` WS in one handler. No gzip, no caching, no HTML rewriting.

**B3 — SSH transport**: system `ssh` binary, injectable (`sshBinary` — the test seam). Control options `-o ControlMaster=auto -o ControlPath=~/.puddle/cm-%r@%h:%p -o ControlPersist=10m` (note: if socket-path length (~104 bytes) bites, switch to `%C` and amend SPEC in the same commit). Master open: `ssh <ctl> user@host true` with `stdio: 'inherit'` (password/2FA typed at most once). `exec` via `sh -c`, `readFile` via `cat`, `isAlive` via `-O check`; `dispose()` never `-O exit` (ControlPersist is deliberate). Tunnel: `-N -L <freeport>:127.0.0.1:<remotePort>`, readiness by TCP poll (never parse ssh output), reconnect loop keeps the same local port (repoint target ref if stolen), emits `tunnel-down`/`tunnel-up` events. Ctrl-C in `cli/run.ts`: first SIGINT → `cockpit.stop()` + "sessions keep running on <host>"; second → hard exit.

**B4 — Windows client degradation**: `platform` injectable; on win32 omit all `Control*` options, warn once ("you may be asked to authenticate more than once; an SSH key avoids the prompts"), and batch bootstrap steps into as few `sh -c '… && …'` execs as possible (good hygiene everywhere).

**B5 — bootstrap + discovery order** (start and connect alike): (1) `readFile(~/.puddle/config.json)` → daemon port, default 7434, `--remote-port` overrides; (2) `readFile(~/.puddle/token)`; (3) both missing → `bootstrap.install()` (runs the embedded `install.sh` over the Transport — see C2), then poll for the token file (≤15 s); (4) tunnel (remote only); (5) handshake. Version detection: `readlink ~/.puddle/bin/current` first; a responding daemon with no symlink = dev install, skip installation, let the handshake govern.

**B6 — handshake** (`decideHandshake` pure): same major → proceed silently. Daemon older → print live-session count (`GET /api/sessions`, filtered client-side to starting/running/waiting_input), re-run bootstrap, restart, `waitForHttp`, re-handshake must match; `--no-upgrade` aborts with the count. Daemon newer → refuse with exact `npm install -g @puddle-code/cli` line. 401 on handshake → re-read token once, then a clear error.

**B7 — attach**: resolve session by exact id or unique prefix (ambiguous → list candidates); `ws` client → `{t:'auth'}`, `{t:'attach', session, term, cols, rows}`; replay then live output verbatim to stdout; raw-mode stdin → `{t:'stdin'}` except **Ctrl-] (0x1d) detaches** (banner on stderr at attach); SIGWINCH → `{t:'resize'}`; `{t:'exit'}` → restore TTY, print code. Streams injected for tests; non-TTY stdin is an upfront error. Remote attach opens master + ephemeral tunnel.

**B8 — status / logs / upgrade**: `status` renders version + host + session table from `/api/version`, `/api/host`, `/api/sessions`. `logs` reads files over the Transport (`tail -c`/`tail -f` of `~/.puddle/logs/<sid>/<term>.log`; daemon logs via `journalctl --user -u puddled` / launchd log path) — **never** via WS attach (a gateway attach resizes the live PTY, `ws/gateway.ts:102`). `upgrade` = unconditional bootstrap re-run with interruption count printed.

**B9 — browser open**: `open`/`xdg-open`/`start` per platform; failure non-fatal, URL always printed. URL: `http://localhost:<port>/` + `?host=user@host` (SSH mode only) + `#token=<hex>`.

## Workstream C — packaging & release (full pipeline)

**C1 — `scripts/build-tarball.mjs`** (new; root `package.json` gains `build:tarball`; `esbuild ^0.25` as root devDependency, JS API, no config files):
- esbuild-bundle daemon → `stage/daemon/puddled.mjs`: `platform: 'node'`, `target: 'node22'`, `format: 'esm'`, externals `['node-pty','better-sqlite3','bufferutil','utf-8-validate']`, `createRequire` banner, `define: __PUDDLED_VERSION__`, no minify/sourcemap.
- Copy + prune natives from live `node_modules` (`fs.cp` with `dereference: true` — preserves the spawn-helper exec bit): node-pty keeps `package.json`, `lib/*.js`, and whichever of `build/Release/` or `prebuilds/<platform>-<arch>/` exists (drops 58 MB of win32 prebuilds, `deps/`, `third_party/`); better-sqlite3 keeps `package.json`, `lib/`, `build/Release/better_sqlite3.node` (drops 9.8 MB `deps/`); plus `bindings`, `file-uri-to-path` whole. Fail loudly if the native binary is absent.
- Node runtime: pinned `NODE_RUNTIME_VERSION` constant (single source of truth, e.g. 22.21.1) — download official tarball, verify against `SHASUMS256.txt`, cache under `node_modules/.cache/`, ship **only `bin/node`**.
- Tarball tree `puddled-v<v>/{puddled (sh launcher: exec "$dir/bin/node" "$dir/daemon/puddled.mjs" "$@"), VERSION, LICENSE, bin/node, daemon/{puddled.mjs, node_modules/…}}`. Current platform only (`process.platform`/`process.arch` — never cross-compile; no `--platform` flag on purpose).
- **Smoke test before tar**: run staged `puddled --version`, assert output (catches broken bundle/banner/wrong-ABI natives).
- Tar via system `tar -czf` with `COPYFILE_DISABLE=1` on darwin; write `<name>.tar.gz.sha256` in `sha256sum -c` format. Expected size ~27–34 MB gz per platform (Node binary ≥95 %). `.gitignore` gains `/dist-release/`.

**C2 — `scripts/install.sh`** (new; POSIX sh, curl-pipeable, **the single bootstrap implementation** — the CLI bundles it as text via esbuild `loader: {'.sh': 'text'}` and pipes it over the ssh master: `ssh host 'sh -s -- --version <v> …'`). Flags: `--version`, `--tarball <path>`, `--sums <path>`, `--repo owner/repo` (env `PUDDLE_REPO`; committed template holds `@@REPO@@`, release job seds in `${{ github.repository }}` — no owner slug in the repo, per conventions), `--no-supervisor`, `--no-start`, `--force`. Behaviour: `uname -sm` detect (musl/Alpine → clear refusal) → fetch from `github.com/<repo>/releases/download/v<v>/…` or accept `--tarball` → `sha256sum -c`/`shasum -a 256 -c` → extract `--strip-components=1` into `~/.puddle/bin/versions/<v>/` → smoke `puddled --version` → `ln -sfn` flip `current` → write `~/.puddle/bin/puddled` as a **two-line exec wrapper** (not a symlink — `dirname $0` must land in `current/`) → supervisor (first match: systemd user unit heredoc with `ExecStart=%h/.puddle/bin/current/puddled`, `Restart=always`, `WantedBy=default.target`, `enable --now` + `loginctl enable-linger` (linger failure = warning, not abort); launchd plist heredoc with `KeepAlive`, `$HOME` expanded, `launchctl bootstrap` falling back to `load -w`, `kickstart -k` on upgrade; nohup+pidfile fallback with the SPEC-mandated warning) → status line. Idempotent; upgrade/rollback = same script, different `--version`.

**C3 — scp fallback + dev override**: CLI downloads tarball + SHA256SUMS to client-side `~/.puddle/cache/` (pruned to last 2 versions), verifies locally, `scp -o ControlPath=…` both to host `~/.puddle/cache/`, runs `install.sh --tarball … --sums …`. `--tarball <path|url>` / `PUDDLE_TARBALL` env skips GitHub entirely (local path: checksum optional with printed warning) — this is how acceptance testing works before any release exists.

**C4 — `.github/workflows/release.yml`** (new): on tag `v*`; matrix `{ubuntu-22.04 → linux-x64, ubuntu-22.04-arm → linux-arm64, macos-13 → darwin-x64 (deprecation risk noted in a comment), macos-14 → darwin-arm64}` — ubuntu-22.04 deliberately (glibc 2.35 floor); each: `pnpm install --frozen-lockfile` → `build-tarball.mjs --version ${GITHUB_REF_NAME#v}` → upload artifact. Release job: concatenate `.sha256` fragments → `SHA256SUMS`, sed `@@REPO@@` in install.sh, `gh release create` with tarballs + SHA256SUMS + install.sh. npm-publish job: gated on `NPM_TOKEN`, `npm publish --provenance --access public -w packages/cli`.

**C5 — CLI package build**: `packages/cli/scripts/copy-web-assets.mjs` (the daemon's, moved, retargeting `web/dist → cli/dist/public`); build = `tsc -b` for dev + an esbuild publish bundle (`dist/index.js`, ESM, `ws` + `@puddle/shared` + install.sh text bundled in, externals only `bufferutil`/`utf-8-validate`, defines for CLI version/pinned daemon version/repo slug from the `repository` field). package.json: name **`puddle-cli`**, bin `"puddle": "./dist/index.js"`, `files: ["dist"]`, `repository`, drop `private` when publishing starts. ~4–5 MB npm tarball (Monaco dominates).

**C6 — CI updates** (`.github/workflows/ci.yml`): embed assertion becomes `test -f packages/cli/dist/public/index.html`; add a tarball smoke step on the ubuntu job (runs `build-tarball.mjs`, which self-tests `puddled --version`) so packaging can't rot between releases.

## Test strategy & verification

- **Pure unit** (vitest, new `packages/cli/vitest.config.ts`): args parsing; `decideHandshake` (4 states × `--no-upgrade`); browser-URL construction; static path confinement (`..`, encoded traversal, SPA fallback); guard predicates; daemon config-migration matrix (fresh → 7434; old 7433 → migrated once; `configVersion:2` + 7433 → kept; custom 7500 → kept); web `rewriteTerminalUri`; host-param clear-on-local-boot.
- **Integration** (`packages/cli/test/`, devDep `@puddle/daemon: workspace:*`): real daemon via `startDaemon({home: tmpdir, port: 0})` (pattern from `packages/daemon/test/e2e.test.ts`) + CLI UI server in front — assert token-authed `/api/version` proxies, 401 without, 403 on bad Host at the CLI layer, tokenless static index, `/ws` upgrade splice with a live `ws` client, streaming unbuffered, clean `stop()`.
- **SSH via fake shim**: `test/helpers/fake-ssh.mjs` interpreting argv (master → exit 0; `-O check` per control file; `sh -c` runs locally against a temp fake host home; `-N -L` runs a real `node:net` TCP forwarder) — exercises full `connect` incl. readlink version detect, tunnel, handshake, reconnect-after-kill, with zero real ssh. Windows degradation asserted as "spawn argv contains no `Control*`" with injected `platform: 'win32'`.
- **Attach**: injected PassThrough streams against a fake-adapter daemon session; replay bytes, stdin frames, 0x1d detach.
- **Packaging**: CI tarball smoke step (staged `puddled --version`); manual: `pnpm build:tarball && puddle connect user@host --tarball dist-release/…` per `docs/acceptance/phase-6.md`.
- **End-to-end manual** (acceptance doc): `puddle start` from a plain shell (per CLAUDE.md, never from inside an agent session) → cockpit at `localhost:7433`, create a session, attach from a second terminal, Ctrl-] detach; remote flow against a real host or a local VM.


## Outcome notes (post-implementation, 2026-07-14)

- Landed as planned, with these deviations decided during the build:
  - `ControlPath=~/.puddle/cm-%C` from the start (not `%r@%h:%p`) — the Unix
    socket-path cap is real; SPEC §10 updated in the same commit.
  - `installedVersion` probes by `readlink bin/current` only (a `--version`
    exec probe of an unknown binary is unsafe by design); SPEC §10 updated.
  - Host paths in shell commands are `"${PUDDLE_HOME:-$HOME/.puddle}"`
    expressions so local runs honour PUDDLE_HOME; scp destinations are
    home-relative because sftp does no remote shell expansion.
  - Found and fixed a latent daemon bug while adding the config migration:
    zod's `.partial()` fills `.default()` values, so `PATCH /api/config`
    reset every omitted setting to its default. The patch schema is now
    built from default-free field validators.
  - `logs` disambiguates a single positional by `@` (host) vs anything else
    (session) — with both positionals optional there is no sniff-free rule.
